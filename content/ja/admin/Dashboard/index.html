Pulsarダッシュボード

Pulsarダッシュボードは、ユーザが全てのトピックの現在の統計情報を表形式で閲覧できるウェブアプリケーションです。

ダッシュボードはPulsarインスタンス (複数のクラスタに跨る) 内の全てのBrokerから統計情報をポーリングし、全ての情報をPostgreSQLデータベースに保存するデータコレクタです。

収集したデータのレンダリングにはDjangoというウェブアプリケーションフレームワークが使用されています。

インストール

ダッシュボードを使用する最も簡単な方法は、Dockerコンテナ内でそれを実行する事です。Dockerイメージを生成するためのDockerfileが提供されています。

Dockerイメージを生成するには次のようにします:

$ docker build -t pulsar-dashboard dashboard

ダッシュボードを実行するには次のようにします:

$ SERVICE_URL=http://broker.example.com:8080/
$ docker run -p 80:80 \
  -e SERVICE_URL=$SERVICE_URL \
  pulsar-dashboard

ここではPulsarクラスタのサービスURLを1つだけ指定する必要があります。これだけで、内部的にコレクタは存在している全てのクラスタとメトリクスを取得する必要のあるBrokerを把握できます。ダッシュボードをスタンドアローンモードで実行しているPulsarに接続している場合、URLはデフォルトでhttp://localhost:8080になります。

Dockerコンテナが起動すると、ウェブダッシュボードにはlocalhostまたはDockerが使用しているホストからアクセスできます。

既知の問題

現時点ではPulsarの認証はサポートされていません。ダッシュボードのデータコレクタは認証関連のいかなるデータも渡さず、Pulsar Brokerが認証を要求している場合にはアクセスが拒否されてしまいます。
